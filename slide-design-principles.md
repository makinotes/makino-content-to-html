# Slide Design Principles — 排版设计原则

PPT 排版的核心规则。生成 slides/deck 时必读。与 slide-types.md（结构规范）和 format-specs.md（提取策略）并列，是 content-to-html 的第二根支柱。

---

## 1. 一页一信息（One Slide, One Message）

- 标题是结论，不是主题词。"市场增长12%" ✅ / "市场概览" ❌
- 标题最多 15 字，2 行，主动语态
- "和"测试：标题里有"和"就该拆成两页
- 60 秒规则：60 秒讲不完的内容太多，拆页
- Ghost Deck 测试：只读所有页标题，能讲完整个故事

---

## 2. 三分法布局（Rule of Thirds）

将 1280x720 画布分为 3x3 九宫格。主要内容放在四个交叉点，不要全部居中。

### Zone 比例

| Zone | 占比 | 像素(720px) | 内容 | 对应 5-layer |
|------|------|-----------|------|-----------|
| Header | ~8% | 60px | 标题 + section label | hdr(40px) + sub(20px) |
| Content | ~86% | 616px | 核心内容，flex 填满 | body(616px usable) |
| Footer | ~4% | 32px | 来源 + 页码 | ft(32px) |

注意：这里的 Zone 比例基于 slide-types.md 的 5 层结构精确计算，不是通用 PPT 指南的 15-18/72-75/8-10%。我们的 header 更紧凑（8% vs 15%），content 区更大（86% vs 72%）。

### 12 列网格

| 布局 | 列分配 | 适用 |
|------|--------|------|
| 全宽 | 12/12 | 图表、大图 |
| 文字+图 | 4+8 (40/60) | 文字主导 |
| 图+文字 | 7+5 (60/40) | 图片主导 |
| 3 列卡片 | 4+4+4 | 三项并列 |
| 4 列卡片 | 3+3+3+3 | 四项并列 |

**永远不要 50/50 分割** — 等分布局缺乏视觉层次。

---

## 3. 8px 间距体系

所有 padding/margin/gap 必须是 8px 的倍数：

| 间距 | 用途 |
|------|------|
| 8px | 紧耦合（图标与标签、列表缩进） |
| 16px | 卡片最小内距 |
| 24px | 卡片舒适内距、卡片间距 |
| 32px | 内容区块间距 |
| 48px | 大区域分隔 |
| 64px | slide 外边距 |

**内距 <= 外距规则（格式塔邻近性）**：卡片内 padding 必须 <= 卡片间 gap。内距 24px 则 gap >= 24px。违反会让卡片融入背景。

---

## 4. 留白设计

**15-20% 规则**：slide 至少 15-20% 面积为空。做不到就删内容。

**留白分两种**：
- 设计留白：围绕焦点元素，引导视线停留
- 意外留白：元素间未分组的空隙，卡片内容稀疏时出现

**让留白看起来是设计的**：
1. 四边等距 margin
2. 相关元素物理靠近，组间用留白分隔
3. 卡片内容少时**缩小卡片高度**而非留内部空白
4. 全出血图片：让图铺满 slide 边缘，文字放在图片负空间里

---

## 5. 列表布局（不要从上到下堆）

| 项目数 | 推荐布局 | 说明 |
|--------|---------|------|
| 3 项 | **横向 3 列** | 每列：图标 + 标题 + 1-2 行描述 |
| 4 项 | **2x2 卡片网格** | 每卡：数字徽章 + 标题 + 一行描述 |
| 5 项 | **横向时间轴/箭头条** 或 3+2 不对称 | 序列用连接器，非序列用 3 上 2 下 |
| 6+ 项 | **拆成两页** 或分 3 组 | 超过认知负荷上限 |

**6x6 规则**：最多 6 条 bullet，每条最多 6 个词。

**数字编号列表设计**：
- 数字徽章：36-48px，accent 背景色，白字
- 标题：16-18px bold，紧邻数字
- 描述：14px regular，标题下方，最多 2 行
- 项目间用分隔线或微妙边框（不只是留白）

---

## 6. 卡片网格设计

### 卡片最低内容

卡片不能只有标题。最低内容 = 图标或数字 + 标题 + 1-2 行描述。

### 稀疏内容处理

| 情况 | 做法 |
|------|------|
| KPI 数字（每卡只有一个数） | 数字放大到 48-72px，标签缩小到 12px。大数字本身就是内容 |
| 概念卡（每卡只有标题+一行） | 缩小卡片高度，内容垂直居中，不要让卡片撑满 body |
| 只有一个词的卡片 | 不要用卡片。改用横向 stat 条（大数字 + 标签内联） |

### 卡片内部层次

```
accent/icon 顶部
↓ 标题/标签
↓ 值/描述
```

### 避免"卡片漂浮"

- 卡片网格区域背景略异于 slide 背景
- 或用左边框 accent（4px solid）替代全边框
- 或用发丝线分隔标题区和卡片区

---

## 7. 图文搭配

### 分割比例

| 类型 | 比例 | 适用 |
|------|------|------|
| 图片主导 | 60% 图 / 40% 文 | 视觉叙事 |
| 文字主导 | 40% 图 / 60% 文 | 论述+配图 |
| 全出血图 | 100% 图 + 文字叠加 | 震撼开场 |
| **永远不要** | 50/50 | 等分争夺焦点 |

### 图片位置

- 图右文左 = 传统阅读流（西方/中文均适用）
- 图左文右 = "揭示"结构（图设上下文，文做解释）
- 图裁切到 slide 右边缘出血 = 创造纵深

### 文字叠加图片

- 不要在复杂图片上直接放深色文字
- 用半透明遮罩（60-70% 不透明度）或模糊或色彩滤镜
- 对比度最低 4.5:1

## 8. 数据呈现

### 选型

| 场景 | 格式 |
|------|------|
| 一个数字就是核心信息 | **Stat 大字**（48-72px 数字 + 标签） |
| 时间趋势 | 折线图 |
| 跨类别比较 | 柱状图 |
| 占比 | 饼图（最多 3 段，否则用柱状） |
| 精确值查询 | 表格 |
| 排名 | 横向柱状图 |

### 图表布局

图表占内容区 60-70%，文字批注占 30-40%（放在图表右侧或下方）。批注写一句洞察，不是标签。

### 图表简化（删掉这些）

- 网格线（用微妙辅助线或删除）
- 图例（直接标注数据系列）
- 3D 效果（永远删）
- 渐变填充
- 超过 4 种颜色
- 标题已说明单位时删轴标题

### 表格增强

- 斑马行（交替浅灰）
- 表头：bold + accent 背景 + 白字
- 数字右对齐，文字左对齐
- 删除所有竖线（用列间距替代）
- 高亮关键列/行（accent 色或 bold）

---

## 9. 图标与装饰

### 图标使用规则

- 图标 + 文字标签必须成对（不能只有图标）
- 一套 deck 一个图标风格（全线框 / 全填充 / 全双色，不混搭）
- 尺寸：内联 24px、卡片头 32px、Hero 48px
- 单色（跟随 deck 配色），不要彩色图标

### 装饰线

| 元素 | 用途 | 规格 |
|------|------|------|
| 卡片左边框 | 分类指示 | 4px solid accent |
| 标题下划线 | 章节分隔 | 2-3px, 40-60px 宽, accent |
| 标题区/内容区分隔线 | 区域划分 | 1px, 10% opacity |
| Slide 全边框 | **避免** | 看起来像模板不像设计 |

## 10. 内容→布局映射

完整决策树和参数速查见 `format-specs.md` "提取后：内容决定布局" 段。此处仅列设计层面的通用规则。

### 通用规则

- **Action Title 必须是结论句**，不是主题词
- **单张元素 ≤6 个**，超过拆分或聚合
- **正文 ≥18px**，低于此后排不可见
- **全 deck ≤5 种布局**，多了视觉混乱
- **全 deck ≤4 种颜色**，重点色只有 1 种
- **表格 ≤6 列**在正文中，超过移至 Appendix

## 硬编码数字速查

| 参数 | 值 |
|------|-----|
| Slide 外边距 | 48-64px |
| 标题区高度 | 15-18% slide 高度 |
| 标题字号 | 28-36px |
| 正文字号 | 18-24px，不低于 18px |
| 标签字号 | 12-14px |
| 卡片内距 | 24px 最小，32px 舒适 |
| 卡片间距 | >= 卡片内距 |
| 间距体系 | 8px 倍数 |
| 最大颜色 | 3（主色+辅色+accent） |
| 最大字体 | 2（或 1 家族不同字重） |
| 每页最多 bullet | 6 条，每条最多 6 词 |
| 图表占比 | 60-70% 内容区 |
| 图文分割 | 40/60 或 60/40，不要 50/50 |
| 最小留白 | 15-20% slide 面积 |
